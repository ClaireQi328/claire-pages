一、CBOM 定义与报价中的地位

1.1 术语澄清

"CBOM" 在制造业中存在双重含义，需首先区分：

缩写全称含义适用场景
CBOMCost Bill of Materials成本物料清单，包含成本信息的BOM视图报价、成本核算、财务分析
CBOMConfigurable Bill of Materials可配置物料清单，基于规则动态生成BOM按订单配置（CTO）、大规模定制

本报告聚焦 Cost Bill of Materials（成本BOM），即以成本视角组织和管理物料清单的专用视图。

1.2 报价流程中的核心角色

CBOM 是连接工程设计与商业报价的关键桥梁：

客户 RFQ（询价单）
↓
技术分析 → EBOM
↓
BOM 转换：EBOM → MBOM → CBOM
↓
成本卷积 Cost Roll-up
↓
内部审批（利润/风险/产能）
↓
报价提交

CBOM 的四大核心价值：

成本透明化
每个组件的材料、人工、间接费用清晰呈现
定价决策支持
基于成本结构确定报价策略
快速响应
客户需求变化时快速重算成本
多方案对比
不同供应商/替代料/工艺的成本模拟

二、BOM 类型体系全景对比

维度EBOM（工程BOM）MBOM（制造BOM）CBOM（成本BOM）PBOM（计划BOM）
视角产品如何设计产品如何制造产品成本构成采购/生产计划
所有者工程/研发部门生产/制造部门财务/成本工程部门采购/计划部门
核心关注功能与形式工艺与工序成本要素与费率供应与交付
数据来源CAD / PLM工艺路线 / 工装成本中心 / 费率表采购 / 库存系统
包含内容零件规格、公差、材料工装、工序、损耗因子材料单价、人工费率、间接费率采购提前期、MOQ、安全库存
转换关系源头← EBOM← MBOM + 成本数据← MBOM + 供应信息
关键联系：CBOM 的物料结构通常来源于 MBOM，但增加了成本维度。EBOM → MBOM 的转换涉及工艺路线（Routing）的定义；MBOM → CBOM 的转换涉及成本中心（Cost Center）和活动类型（Activity Type）的映射。

三、CBOM 核心数据结构

3.1 头表（Header）

字段说明
BOM_ID唯一标识
PART_ID / MATERIAL_CODE顶层物料编码
BOM_VERSION版本号
EFFECTIVE_DATE / VALID_DATE生效日期 / 有效期
STATUS草稿 / 审批中 / 已发布
PLANT_ID工厂代码
COSTING_TYPE标准成本 / 实际成本 / 估算成本

3.2 明细行（Line Items）

字段说明
COMPONENT_ID子件物料编码
QUANTITY用量 / 单位用量
UNIT_PRICE单价
CURRENCY币种
COST_CATEGORY直接材料 / 间接材料 / 替代料
SUPPLIER_ID供应商信息
LEAD_TIME采购提前期
SCRAP_RATE损耗率
OVERHEAD_RATE间接费率 / 管理费率
TOTAL_COST总成本 = 单价 × 用量 × (1+损耗率) + 间接费

3.3 成本卷积算法（Cost Roll-up）

产品总成本 = Σ(子件成本) + 制造费用 + 管理费用 其中： 子件成本 = 单价 × 用量 × (1 + 损耗率) × 汇率 制造费用 = Σ(工时 × 活动费率) 管理费用 = 材料成本 × 间接费率

四、SRM 系统中的 CBOM 实现

4.1 存储与版本管理

  • 存储方式：关系型数据库，经典 Header-Detail 主子表结构
  • BOM 变体：同一产品可有多个 CBOM 版本（不同供应商方案、汇率场景、产能假设）
  • 版本控制：效果日期管理（Effectivity）+ 工程变更号（ECN）
  • 审计追踪：记录每次变更的历史（变更人、内容、原因）

4.2 审批流程

BOM 创建/变更
↓
成本工程师审核（技术可行性）
↓
采购经理审核（供应商/价格合理性）
↓
财务总监审核（利润率/预算合规）
↓
最终审批发布

六、行业对标分析

特斯拉 Tesla
垂直整合 + 制造创新驱动

核心策略

  • 高度垂直整合：电池、电机、软件大量自研自产，传统车企外包 80% 零件，特斯拉反其道
  • Gigacasting 一体化压铸：大型压铸件将零件数量减少 50%+，根本性改变 BOM 结构
  • 直接与原材料商签约：绕过中间商，如直接与锂矿商签长期合同

关键数据

$112/kWh
电池成本（2021），行业均值 $132
30-40%
电池占整车成本比重
50%+
Gigacasting 减少零件数量
CBOM 启示：制造工艺创新会根本性改变 BOM 结构，CBOM 系统需适应快速的工艺变革。直接材料成本占比高，间接费用占比相对低。
字节跳动 ByteDance
ODM 模式 + 数据驱动

核心策略

  • ODM/OEM 为主：硬件产品外包给富士康、歌尔股份等代工厂
  • 集中采购：GPU 等高价值组件通过规模效应降本
  • 数据驱动：自研数据分析系统优化供应链成本

CBOM 管理特点

  • CBOM 管理更多在代工厂端完成
  • 侧重新品导入（NPI）阶段的目标成本管理
  • 硬件 BOM 变更频繁（产品迭代快），需快速成本重估能力
CBOM 启示：科技企业的 CBOM 更多是需求端管理，关注目标成本而非制造过程成本。
丰田 Toyota
精益生产 + 目标成本法标杆

核心策略

  • 目标成本法：从市场可接受价格倒推目标成本，分解到 CBOM 每个组件
  • 消除七大浪费（Muda）：过量生产、等待、运输、加工浪费、库存过剩、不必要移动、缺陷
  • JIT + Kaizen：持续的成本降低活动
  • 供应商协同降本：1990年代起将 TPS 方法论延伸至供应商，共享成本改善成果

供应商关系

  • 长期合作（部分供应商合作超过 30 年）
  • 供应商开发（帮助供应商提升能力、降低成本）
  • 价值工程（VE/VA）协作
  • 要求供应商公开成本结构，进行联合成本分析
CBOM 启示：丰田的 CBOM 强调"应该成本"（Should-cost）分析，目标成本分解是 CBOM 设计的核心逻辑。供应商长期关系保障了成本数据的准确性和时效性。
苹果 Apple
规模效应 + 极致谈判

核心策略

  • 极度规模效应：单一产品巨量采购，极强议价能力
  • 早期供应商介入（ESI）：设计阶段引入供应商，优化可制造性和成本
  • 竞争性报价：同一组件保持 2-3 家供应商，持续施压
  • Design-to-Cost：产品设计阶段就设定成本目标

关键数据

1.8%

CBOM 启示：CBOM 的精确度决定议价能力。苹果构建极其精细的成本 BOM + Should-cost modeling，运输、关税、库存持有成本等全链路隐性成本也纳入 CBOM。
华为 Huawei
自研系统 + 风险成本

核心策略

  • 自研 iProcurement 平台：全流程采购系统
  • 海思 + 鸿蒙：芯片 + OS 双层自研，减少外部依赖
  • 战略备货：制裁前大规模囤积关键元器件
  • 国产替代：推动国内供应链成熟，长期降本
CBOM 启示：CBOM 需考虑供应链韧性成本（地缘政治因素），多币种多区域的成本管理能力不可或缺。

七、最佳实践与系统集成

7.1 CBOM 驱动的自动报价

客户 RFQ 输入
→
BOM 自动解析 (PLM)
→
成本自动卷积
→
多方案成本模拟
→
利润计算 + 报价单生成
→
审批 → 输出

关键技术能力

  • 参数化定价：基于产品配置参数自动计算成本
  • AI/ML 定价引擎：基于历史报价数据训练模型，预测合理报价区间
  • 实时汇率集成：多币种报价时自动应用最新汇率
  • 成本敏感度分析：自动分析各成本要素波动对总成本的影响

7.2 Should-cost Modeling（应该成本建模）

独立估算每个组件的合理成本，验证供应商报价的合理性：

合理成本 = 原材料市场价格 + 加工成本 + 合理利润率 验证逻辑： 供应商报价 ≤ 合理成本 × (1 + 缓冲系数) → 报价合理 供应商报价 > 合理成本 × (1 + 缓冲系数) → 需要议价或更换供应商

推荐工具：aPriori、CostWorks 等专业成本建模软件。

7.5 核心建议

  1. CBOM 是报价的核心数据资产，需提升到战略高度，而非仅作为 ERP 的附属功能
  2. 数据准确性是基础，建立定期数据质量审查机制
  3. PLM-ERP-MES-SRM 深度集成才能实现端到端价值
  4. CBOM 是动态数据，需随市场价格、供应商变化、工艺改善持续更新
  5. 离散制造侧重组件级成本管理，流程制造侧重配方/工艺成本管理

八、与当前制造费聚合表的关联分析

当前选中的 制造费聚合表 正是 CBOM 中制造费用（Overhead）部分的典型实现——覆盖了产能信息、设备费用、能耗费用、报废费用、耗材费用、人员费用六个维度的制造费聚合逻辑。

关键观察

  1. 制造费聚合表 = CBOM 的核心组成部分——它定义了 EP 系统中制造费用的完整计算逻辑（公式已在表中定义）
  2. CBOM + 制造价聚合 = 完整的成本估算——表中"设备制造总费"就是 CBOM 中成本卷积的一个实例
  3. SVC（供应商成本验证）对应 CBOM 中的供应商报价验证——字节跳动的做法（新品 NPI 阶段目标成本管理）可以参考
  4. 建议扩展：当前表缺少物流成本和管理费用（间接费率），完整 CBOM 应覆盖这两项
  5. 丰田 Should-cost 模式：表中已有多个"如计算因子有值则计算；若任意因子为空则支持手动输入"的逻辑，这正是 Should-cost 的弹性设计——有数据自动算，没数据人工填，两者并存
落地建议：将当前制造费聚合表纳入 CBOM 体系时，建议补充以下字段以实现完整的成本卷积：

• 物流成本（运输、包装、关税）
• 管理费用（间接费率、管理费率）
• 质量成本（退货、返工、报废的全链路成本）
• 汇率因子（多币种报价支持）
