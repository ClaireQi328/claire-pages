文档说明 Module Documentation

模块概述 Overview

本模块为 E2E-SCM 供应链管理平台的「生产制造」子系统，覆盖从排产、执行、质量到设备的全流程管理。包含以下核心功能：

  • 生产总览 - 实时监控产线状态、OEE、直通率、日产量等关键 KPI
  • 排产管理 - 工单甘特图、产能利用率分析、优先级排序，支持 SPT/EDD/CR 排产算法与拖拽调度
  • 质量管控 - IQC/IPQC/OQC 全链路检验、SPC 过程控制、Pareto 缺陷分析
  • SPC分析 New - X-bar/R 控制图生成器，含 Western Electric Rules 检测与过程能力 Cpk 计算
  • OEE计算 New - 设备综合效率计算器，含可用率/性能率/质量率拆解与 Pareto 损失分析
  • 设备监控 - 12台设备实时状态、维护保养日历、MTBF/MTTR 指标
  • 外协管理 - 供应商评估、外协订单跟踪、自产 vs 外协质量对比
  • 工艺管理 - 工艺路线、BOM 维护、SOP、ECN 管理（规划中）

SPC 控制图说明 核心算法

SPC（Statistical Process Control）统计过程控制模块实现了完整的 X-bar & R 控制图生成与分析：

  • 控制限计算：UCL = X-double-bar + A2 * R-bar, LCL = X-double-bar - A2 * R-bar
  • A2 系数：基于子组大小 n 的标准常量（n=5 时 A2=0.577）
  • Western Electric Rules：4 条判异准则，自动检测过程异常模式
  • 过程能力：Cp = (USL-LSL)/(6σ), Cpk = min(Cpu, Cpl)
  • 能力评级：Cpk ≥ 1.33 能力充分 | 1.0 ≤ Cpk < 1.33 勉强 | Cpk < 1.0 不足

OEE 设备综合效率 核心算法

OEE（Overall Equipment Effectiveness）是衡量设备效率的黄金标准：

  • 可用率 Availability = (计划运行时间 - 停机时间) / 计划运行时间
  • 性能率 Performance = (实际产量 × 理想节拍) / 实际运行时间
  • 质量率 Quality = 良品数 / 总产量
  • OEE = Availability × Performance × Quality
  • 世界级基准：OEE ≥ 85%（可用率 ≥ 90%, 性能率 ≥ 95%, 质量率 ≥ 99%）

排产优化算法

支持三种经典调度规则：

  • SPT（Shortest Processing Time）- 最短加工时间优先，最小化平均流程时间
  • EDD（Earliest Due Date）- 最早交期优先，最小化最大延迟
  • CR（Critical Ratio）- 关键比率 = (交期-当前时间)/剩余加工时间，CR 越小越紧急

技术实现

  • 纯前端实现，无外部依赖，单 HTML 文件自包含
  • SVG 渲染控制图，支持动态交互
  • 所有计算均在浏览器端 JavaScript 完成
  • 拖拽排产基于 HTML5 Drag & Drop API
