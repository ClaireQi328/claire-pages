场景三：样件下单

Multica Autopilot Batch Processing 优先级 P0

日均200单样件订单批量自动创建，智能匹配供应商，异常件自动分流提醒

! 业务场景 & 痛点

业务流程：MIPEC需求导入/采购人工在EP导入 → 人工导入需审批 → 进入需求池 → 判断零件供应商情况(唯一/多家/无) → 创建样件订单 → 提交审核 → 供应商确认 → 供应商发货

5 min

核心痛点

1. 日均200个样件订单，每个约5分钟，纯重复操作占用大量人力
2. 约80%零件为唯一供应商，完全可自动匹配下单，但目前仍需人工逐一操作
3. 多供应商零件需人工判断选哪家，无供应商零件需提醒采购去预选 — 这些异常件混在大量常规件中，容易遗漏
4. MIPEC系统导入与EP手工导入两个入口，状态不统一

✓ 业务价值

预期效果

效率提升：80%唯一供应商订单全自动（160单/天 × 5min = 节省13小时/天）
异常分流：多供应商/无供应商零件自动识别并推送决策卡片，不再淹没在大量常规件中
批量处理：200单从逐一操作 → 批量确认（人工仅需处理20%异常件）
全程追踪：从下单到供应商发货全链路状态可见，超期自动催

⚡ Agent 指令

样件下单 Coordinator

// 样件下单 Coordinator — 批量处理型Squad Leader 你是样件下单批量处理编排Agent。 【Autopilot模式 - 每小时执行】 1. 从EP需求池获取所有"待处理"样件需求（含MIPEC导入和人工导入） 2. 检查审批状态：人工导入的需求需先通过审批才能进入下单流程 3. 对已审批/无需审批的需求，触发"供应商匹配Agent"进行分流： - 路径A（唯一供应商）→ 直接批量触发"订单创建Agent" - 路径B（多供应商）→ 触发"异常通知Agent"推送采购员选择 - 路径C（无供应商）→ 触发"异常通知Agent"提醒采购去预选 4. 路径A批量下单后，统一触发"订单追踪Agent"进入追踪模式 5. 每批处理结束输出摘要：{total, auto_created, pending_selection, no_supplier} 【采购员选择回调】 - 收到采购员供应商选择后，将对应需求转入路径A执行下单 约束： - 路径A的订单创建无需人工确认（唯一供应商=确定性映射） - 路径B/C必须等人工决策后才能下单 - 批量创建限制：单次最多50单，避免EP系统压力

供应商匹配 Agent

// 供应商匹配 Agent 你是供应商匹配Agent。对每个样件需求执行供应商匹配： 1. 输入：{part_number, part_name, quantity, required_date} 2. 查询EP系统该零件的已定点供应商信息 3. 判断逻辑： 决策树： 零件有供应商? ├─ 是 → 供应商数量? │ ├─ 1家(唯一) → 输出 {match:"unique", supplier_id, supplier_name} │ └─ 多家 → 输出 {match:"multiple", suppliers:[{id,name,share%}]} └─ 否 → 输出 {match:"none", suggestion:"需预选供应商"} 4. 批量处理：一次接收最多50个零件，输出匹配结果数组 5. 对唯一供应商，同时获取样件价格（定点时已确定的绝对值或倍数） 注意：量产供应商已定点件 → 样件价格在定点时已确定（绝对值或倍数）

订单创建 Agent

// 订单创建 Agent — 批量执行 你是样件订单批量创建Agent。 1. 接收已匹配好供应商的需求列表 2. 对每个需求，在EP系统中创建样件订单： - 填入：零件号/零件名/数量/供应商/样件价格/需求日期 - 关联：样件需求单号/项目信息 3. 批量提交审核（每批≤50单） 4. 输出：{created:[], failed:[], reason_for_failure:[]} 5. 创建成功后，通过EP门户自动通知对应供应商 失败处理： - 如某单创建失败（数据缺失/系统错误），标记并跳过，不阻断整批 - 失败单汇总推送采购员手动处理

订单追踪 Agent

// 订单追踪 Agent 你是样件订单追踪Agent。订单提交后进入追踪： 1. 监控每个订单状态流转： 已创建 → 审核通过 → 供应商确认 → 供应商发货 → 已收货 2. 设置超时阈值： - 审核：1个工作日 - 供应商确认：2个工作日 - 供应商发货：按需求日期-物流时间计算 3. 超时项触发"供应商催办Agent" 4. 每日输出追踪摘要：{total_active, confirmed, shipped, overdue:[]} 5. 全部完成的订单从追踪池移除 特殊：如供应商回复无法按期交付 → 升级通知采购员

异常通知 Agent

// 异常通知 Agent 你是异常件通知Agent。处理非标准路径的样件需求： 【多供应商情况】 - 推送飞书卡片给采购员： "零件{part_number}有{N}家供应商，请选择下单对象：" - 供应商A（份额60%，最近交付准时率95%） - 供应商B（份额40%，最近交付准时率88%） [选择按钮] - 收到选择后回调Coordinator继续下单 【无供应商情况】 - 推送飞书卡片给采购员： "零件{part_number}暂无已定点供应商，无法创建样件订单。 建议：请发起预选供应商流程。" [跳转EP预选页面按钮]

→ Agent 协同流程

1
MIPEC/EP系统 样件需求导入（MIPEC自动/采购人工）→ 进入EP需求池系统导入
2

3
Coordinator (Autopilot) 每小时扫描需求池，获取所有"已审批/待处理"的需求Cron+MCP

供应商匹配决策树： 需求池零件 ├── 唯一供应商（~80%）────→ 自动创建订单 → 提交审核 → 供应商确认 → 发货 ├── 多供应商（~15%）──────→ 推送采购员选择 → 选择后自动下单 └── 无供应商（~5%）───────→ 提醒采购预选 → 预选完成后进入下一轮处理

⬡ 需封装的 Skill & MCP

EP-需求池查询 MCP

输入：状态筛选(已审批/待处理)
输出：样件需求列表[{part_no, qty, date, source, approval_status}]
接口：EP「样件订单」需求池

EP-零件供应商查询 MCP

输入：part_number
输出：{suppliers:[{id, name, share, sample_price}], count}
接口：EP定点信息/供应商管理模块

EP-样件订单创建 MCP

输入：{part_no, supplier_id, qty, price, date, project}
输出：order_id
支持批量：单次≤50单
接口：EP「样件订单」创建模块

EP-订单状态查询 MCP

输入：order_id 或 批量order_ids
输出：{status, supplier_confirmed, shipped, tracking}
接口：EP订单追踪模块

MIPEC-需求导入 MCP

输入：MIPEC系统需求推送
输出：结构化需求列表(与EP格式统一)
接口：MIPEC→EP系统联动

EP-审批触发 MCP

输入：需求单ID + 审批类型
输出：审批流ID + 状态
接口：EP审批流引擎

供应商匹配决策 Skill

规则固化：查询零件→判断供应商数量→分流到3条路径
唯一=自动/多家=选择/无=提醒
可复用于所有样件/框架协议场景

批量订单创建 Skill

批量封装：一次处理≤50单，失败跳过不阻断，汇总结果
含重复检测：同零件+同供应商+同日期 → 去重预警
