CLI & Skill 基础建设

采购业务域 CLI 基础体系搭建、业务能力封装及智能应用模式探索

背景与目标

零件定点全流程涉及 SOR、预选、询报价、物流确认、定点签约等多个分散节点，链条冗长、状态难追踪；采购、DRE、SQE、物流等岗位需求各异，缺乏统一查询入口。单条 CLI 只能完成原子查询，实际业务场景往往需要跨域组合（如某供应商是否可以创建定点申请单需串联多条指令）。

供应商 SOR RFQ 定点申请单

方案

1
定义 17 个 CLI 指令
覆盖供应商、SOR、RFQ、定点申请单四大业务域。采用原子化设计，统一入参规范、标准化出参结构、模块间独立解耦，为上层 Skill 和后续跨域复用提供标准化数据调用基座
2
封装业务 Skill
基于 CLI 组合封装业务 Skill，将多步查询编排为端到端的业务流程。既可独立供业务人员使用，也为后续跨域复用积累可移植的封装模式
