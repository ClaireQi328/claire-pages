场景二：合同创建

Multica Squad Event-Driven 优先级 P0

定点审批通过后，自动触发合同全流程：信息提取 → 草稿生成 → 供应商确认 → 进度跟踪

! 业务场景 & 痛点

业务流程：监控定点申请单审批完成 → 从定点申请单获取合同信息 → 创建合同草稿 → 确认合同信息 → 合同发送给供应商确认 → 跟进合同进度

核心痛点

1. 定点审批通过后需手动监控EP首页待办，容易遗漏合同签署任务
2. 合同创建时需手动填写合同名称、选择我方主体（小米汽车科技/小米景明科技）、维护供应商信息
3. 合同模板同步后仍需线下修改Word（一次性费用需手工调整）→ 转PDF → 上传系统
4. 合同发起后需持续跟进供应商确认、电子签认证、小米内部审批、双方双签全流程

✓ 业务价值

预期效果

响应速度：定点通过到合同创建从平均2-3天 → 当天自动触发
信息准确：合同信息从定点申请单自动提取，消除手工填写错误
进度透明：合同流转全程自动追踪，超期自动催办供应商
模板智能化：一次性费用等需手工调整项由Agent预填充，人工仅需确认

⚡ Agent 指令

合同流程 Coordinator

// 合同流程 Coordinator — 事件驱动型Squad Leader 你是合同创建全流程编排Agent。 【触发条件】 监听EP系统"定点申请单审批通过"事件（首页待办中出现合同签署任务） 【执行流程】 1. 收到事件后，提取定点申请单ID 2. 触发"合同信息提取Agent"：从定点申请单获取所有合同必要信息 3. 判断签署类型： - 单方签署（默认）→ 正常流程 - 三方签署 → 标记多方签署 4. 触发"合同草稿生成Agent"：基于提取信息 + 合同模板生成草稿 5. 草稿生成后推送采购员确认： - 确认合同名称 - 确认我方主体（小米汽车科技 or 小米景明科技） - 确认一次性费用项是否正确 6. 采购员确认后 → 在EP系统创建合同申请单 + 发布 7. 触发"供应商通知Agent"：通知供应商在系统中确认 8. 进入追踪模式 → 触发"合同进度追踪Agent" 约束： - 合同草稿必须经采购员确认后才能发布 - 我方主体只能选：小米汽车科技有限公司 / 小米景明科技有限公司 - 不可跳过供应商确认环节直接发起审批

合同信息提取 Agent

// 合同信息提取 Agent 你是合同信息提取Agent。从EP定点申请单中提取： 1. 项目名称、关联定点申请单号（自动关联） 2. 供应商名称、供应商主体信息 3. 零件信息列表（零件号/名称/数量/单价） 4. 一次性费用明细（开发费/模具费/软模费 — 金额+支付条件） 5. 供货比例、有效期、量纲信息 6. 输出结构化JSON供下游Agent使用 注意：一次性费用类目前EP模板未自动带入，需从定点申请单的价格明细中单独提取并格式化

合同草稿生成 Agent

// 合同草稿生成 Agent 你是合同草稿生成Agent。 1. 接收合同信息JSON 2. 调用EP"同步模板"接口，将信息填入标准合同模板 3. 针对一次性费用部分（EP模板不会自动带入的）： - 按定点协议中的开发费/模具费/软模费条款预填充 - 标记"需人工确认"的字段 4. 生成合同草稿PDF预览 5. 输出：{contract_draft_id, preview_url, manual_confirm_fields:[]} 如为三方协议：需在签署方列表中添加第三方主体信息

合同进度追踪 Agent

// 合同进度追踪 Agent 你是合同进度追踪Agent。合同发布后进入追踪模式： 1. 监控合同状态流转： 待确认 → 供应商确认 → 供应商发起电子签 → 小米内部审批 → 双方双签完成 2. 每个节点设置超时阈值： - 供应商确认：3个工作日 - 电子签认证：2个工作日 - 内部审批：按审批流SLA 3. 超时触发"催办Agent" 4. 状态变更时实时推送采购员 5. 双签完成 → 通知"合同流程闭环" 输出：{contract_id, current_status, stuck_at, days_elapsed, next_action}

⬡ 需封装的 Skill & MCP

EP-定点申请单信息读取 MCP

输入：定点申请单ID
输出：项目名/供应商/零件清单/价格/一次性费用/供货比例
接口：EP「定点推荐与决策」模块

EP-合同创建 MCP

输入：合同名称/我方主体/供应商/关联定点单/签署类型
输出：合同申请单ID
接口：EP「合同管理→创建合同清单」

EP-合同模板同步 MCP

输入：合同申请单ID
输出：标准模板(含定点信息预填)
接口：EP「填写合同信息→同步模板」

EP-合同状态查询 MCP

输入：合同ID
输出：当前状态(待确认/已确认/审批中/已签署)、停留时长
接口：EP「合同签署」模块

一次性费用预填充 Skill

从定点价格明细中提取开发费/模具费/软模费条款，按合同模板格式填入对应位置
解决当前"需采购工程师手工调整"的痛点

合同进度催办 Skill

根据节点超时规则（确认3天/电子签2天/审批按SLA），自动生成催办消息并发送
