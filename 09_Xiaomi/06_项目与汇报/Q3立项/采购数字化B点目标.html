2.2 采购数字化 B 点目标

成本 ✖️ 效率 ✖️ 合规 ✖️ 供应商　|　四层架构 ✖️ 两大闭环

以成本、效率、合规、供应商四维为牵引，实现采购"价格看得清算得准、流程自动化不费力、合规说得清、供应商管得住"；以四层架构为承载、两大闭环为引擎——让采购从"系统记录人决策"的执行工具，升级为"会自己进化的采购大脑"。

沿 ① 采得到信得过 → ② 端到端融数据 → ③ 看得见算得出 → ④ 根因决策回写逐层跃迁，完成 L1（看板辅助）→ L3（常规自治） 的能力升级。
01

为什么是「成本 × 效率 × 合规 × 供应商」

Why these four
供应链全局用 DQCC（交付/质量/成本/合规）四域衡量，但 交付与质量的主战场在制造与计划侧，采购能直接握住的杠杆是四根：成本、效率、合规三轴解决"这单怎么买更优"，供应商维度解决"谁来供、长期供得好不好"——前三者是单次价值，供应商是贯穿全生命周期的资产底盘。
成本 C
价格的第一责任人
决定"花多少钱买"
直接计入毛利——利润底盘。
效率 E
流程的执行中枢
决定"用多少人、多久买到"
人均产能决定能管多大盘子——可扩展性放大器。
合规 C
出海的守门人
决定"能不能合法交付"
漏一项即出口受阻——0/1 资格门槛。
供应商 S
资源的资产底盘
决定"谁来供、供得稳不稳"
绩效/风险/分层贯穿全生命周期——资产基本盘。
四维不是并列，而是乘法关系：合规是 0/1 资格门槛、效率是放大器、成本是利润底盘、供应商是承载前三者的资产底盘。任一维塌陷，其余维度的成果都会清零——这正是用"✖️"而非"+"表达的原因。
02

四维的 B 点目标（做什么）

What to build
成本价格第一责任人 · 利润底盘　|　从量产期事后核价 → 设计期主动锁定
  • 建 Should-cost 标尺 + 成本因子库：从功能/工艺/材料/工时费率倒推目标价，定价窗口前移至设计期（锁定 70-80% 成本结构）
  • A+B 自动卷积：DB 件变更自动触发总成件重算、自动确认、自动刷台账，价格确认从 3-5 天压缩至秒级
  • 成本模拟可视化：不同车型/配置/份额/原材料价格多情景试算，设计与定点前看清成本结构与降本空间，辅助 DFC 评审与选型
  • 大宗行情自动订阅 + 历史价格沉淀，形成可比价格基线，支撑趋势预判与异常报价检测
  • 偏差自动归因（设计变更/材料波动/工艺/份额）并回写因子库，基线随每轮迭代更准
效率流程执行中枢 · 可扩展性放大器　|　从大量人工线上操作 → 机器自动执行、人管例外
  • 框架协议合并为统一台账、合同全生命周期在线统一签署、定点/设变/变价进任务中台统一纳管
  • 样件订单自动化：常规件系统直下、异常件自动识别升级，释放 40% 可自动化单量（≈1 FTE；现状日均 200 单、16.7 小时）
  • 引入采购 Agent（汇信息出草稿、识风险、给建议），人做确认与例外决策；分阶段 Phase 1 半自动 → Phase 2 低风险全自动
合规出海守门人 · 0/1 资格门槛　|　从分散应对、人工举证 → 体系化、数据级自证
  • 12 项法规整合为尽调/碳足迹/电池护照/禁用物质 4 条线，三统一 + 统一问卷做到一次采集、多场景复用
  • 碳核算引擎（因子库×活动数据自动算）+ 多源风险整合（CAMDS/天眼查/穆迪），替代 Excel 手算与未验证数据
  • 整车→Tier1→Tier2→矿山全链路打通，高风险供应商 100% 覆盖，保出海车型合规交付 100%（2027.7 硬节点）
供应商资源资产底盘 · 全生命周期治理　|　从长清单 + 人工筛选 → 资源池 + 绩效 + 标签在线化的动态管理
  • 资源池管理：建品类维度准入资源池，准入/评估/退出机制完整——从"长清单 + 人工筛选"升级为结构化资源池 + 规则准入
  • 绩效评价：多维绩效模型（质量/交付/成本/服务/技术）自动评分——从"定点会信息靠人对齐"升级为客观指标秒级可视
  • 供应商画像：标签在线化，财务/工厂布局/客户等关键属性可检索——从"线下单次全量收集"升级为多条件组合检索 + 持续更新
  • 策略支撑：分层分级模型挂钩采购策略（战略/杠杆/瓶颈/常规）——从"一视同仁"升级为差异化策略 + 份额动态调整
03

四维 ✖️ 四层架构（建什么）

Axes × Layers
四层依赖：① 采得到信得过 → ② 端到端融数据 → ③ 看得见算得出 → ④ 根因决策回写。横轴是四个价值方向，纵轴是四层能力——交叉处即每个方向在每一层要补的料。
　成本 C效率 E合规 C供应商 S
④ 协同决策根因 + 决策 + 回写 DFC 评审机制、偏差→基线回写、成本因子库自迭代 Agent 自治/人确认、任务异常升级、流程自优化 风险尽调闭环、口径冲突校验、标准/方法学随法规升级 绩效驱动份额调整、分层分级自动升降、退出/淘汰机制
③ 分析模型看得见 + 算得出 Should-cost 偏差、报价异常、A+B 卷积、降本机会 智能定价匹配、供应商匹配决策、需求归集 碳核算引擎、风险评分、合规差距自动评估 多维绩效自动评分、风险预警、分层分级模型
② 数据汇聚端到端 + 融合 成本因子库、报价组项、历史价格、CE 核价 框架统一台账、订单主数据、任务中台 法规主数据、排放因子库、供应商合规档案 供应商资源池、绩效档案、标签画像、品类策略库
① 数据采集采得到 + 信得过 大宗行情、CE 目标价直连、BOM/VSM 结构化 MIPEC 需求、多维表格、定点台账 统一问卷、CAMDS/天眼查/穆迪、关键原材料碳数据 注册/准入信息、财务/工厂布局、质量交付履约数据
关键原则：底层夯实，逐层加码。 数据没采可信就别急着上模型，主数据没建好就别急着上决策——四维在每一层"对齐着往上爬"，避免单维冒进。
04

四维 ✖️ 两大闭环（怎么转）

Two loops
⚡ 实时执行闭环
秒-时级 · ①②③ 持续运行
感知数据 → 聚合汇集 → 模型判断 → 输出正常/异常
  • 成本：DB 件变更 → 总成件自动重算 → 刷台账
  • 效率：需求归集 → 智能匹配价格 → 可自动单直接下单
  • 合规：供应商填报 → 多源校验 → 风险分级实时呈现
  • 供应商：质量/交付履约回传 → 绩效自动评分 → 资源池状态实时刷新
▼ 异常落入
🔄 分析进化闭环
日-周-月级 · ③④②① 反向控制
判断/异常 → 根因 → 人机协同决策 → 执行 → 回写基线
  • 成本：偏差归因 → DFC 评审 → 成本因子库更新 → 下一轮用新基线
  • 效率：异常件升级 → 人工处置 → 规则沉淀 → Agent 下次自动处理
  • 合规：风险发现 → 整改/复核 → 标准/方法学升级 → 新法规一键复用
  • 供应商：绩效下滑/风险触发 → 分层分级调整 → 份额/淘汰决策 → 回写资源池
实时执行闭环让四维日常"自己跑"，正常即直接执行、无需人介入；分析进化闭环让每一次异常关闭都沉淀为下一轮更优基线——让经验在系统"长住"，而不是"用一次就过期"。
