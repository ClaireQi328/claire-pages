1报价细项功能说明

📝 备注说明

统计口径：本报告统计的数据范围为活跃报价包（报价细项已上线但未完成发包的RFQ报价包）。

活跃RFQ定义：报价细项上线前、但尚未发包的RFQ，其供应商也可以参与细项拆解。即：只要报价包处于"报价细项已上线 → 待发包"这一阶段，均纳入活跃报价包统计范围。

拆解类型说明："自行拆解"指供应商自行填报的成本构成；"BOM结构"指按物料清单结构拆解；"BOM+VSM"指在BOM结构基础上叠加价值流映射的深层成本穿透。

例外品类：部分品类（如导线、线束等标准化程度高或成本构成简单的品类）暂不强制要求成本拆解，待业务侧制定分品类的拆解标准后另行纳入考核。

🎯 1.1 核心价值

报价管理作为供应链及生产经营的核心环节，当前面临数据割裂、标准不一、分析滞后等痛点，制约业务效率与战略决策落地。报价细项涵盖总成材料明细与制造信息两大板块，前者涉及零部件组合、成本构成、报价有效期等维度，后者关联生产工艺、物料消耗、工时成本等核心数据。

🧮

报价明细拆解 = 把"供应商说了算"变成"数据说了算"

报价总价 = 材料费 + 制造费 + 模具费 + 开发费 + 管理费 + 样件费

❌ 痛点：当前现状

  • ✗ 数据分散于多系统、表格及线下文档
  • ✗ 缺乏统一底层主数据支撑
  • ✗ 数据口径混乱、重复录入
  • ✗ 报价核对人工成本高
  • ✗ 信息遗漏、易引发数据误差

✅ 目标：数据说了算

  • ✓ 报价明细拆解 → 精准比价
  • ✓ 费用逐项对比 → 靶向议价
  • ✓ 成本结构透明 → 科学定价
  • ✓ 从被动接受价格 → 主动分析成本
  • ✓ 消除黑箱，每一分钱花在哪里

零穿透 部分穿透 完全穿透
💡
零穿透（红点）密集分布于底部，覆盖 1~7 家供应商全部区间——无论供应商多少，零穿透都是普遍现象。部分穿透（黄点）集中在 3~6 家供应商的 RFQ，穿透率 20%~50%。完全穿透（绿点）仅出现在供应商数 = 1 的独供场景（如整车玻璃系统、天际屏2.0），且均为自行拆解。供应商 ≥5 的 RFQ 中零穿透仍占绝大多数（如高压导线 DB 7 家供应商全部未拆解），说明：供应商越多 ≠ 竞争越充分 ≠ 透明度越高。
📋 已支持的运行场景
场景一

BOM ❌ · VSM ❌ → 不拆解报价

BOM 和 VSM 数据均不可用，用户不拆解直接进行总价报价。

无 BOM 数据 无 VSM 数据 总价报价
场景二

BOM ❌ · VSM ❌ → 自行拆解报价

BOM 和 VSM 数据均不可用，用户自行拆解 BOM 结构进行报价。

用户手动拆解 自行维护 BOM
场景三

BOM ✅ · VSM ❌ → 按 BOM 结构报价

BOM 数据可用，按照 BOM 结构进行报价，VSM 暂不可用。

系统 BOM 数据 结构化报价
场景四

BOM ✅ · VSM ✅ → BOM+VSM 深层报价

BOM 和 VSM 数据均可用，按照 BOM 结构 + VSM 结构进行深层穿透报价。

系统 BOM VSM 深层穿透 最高精度

2长期运营方向

📐 建议：分品类分级管理

不同品类的成本结构复杂度差异较大，建议按 A/B/C/D 四级分类管理，避免"一刀切"导致低复杂度品类投入产出比失衡。

等级 典型品类 建议拆解要求 建议节奏
A 级 座椅总成、减振器、电池系统、电驱壳体、CCB 镁合金 BOM + VSM 建议立即执行
B 级 内饰件、安全带、门把手、保险杠、紧固件 BOM 结构 建议 3 个月内
C 级 通用螺栓、色漆、结构胶、膨胀胶块 至少自行拆解 建议 6 个月内
D 级 导线 DB、线束总成、蜂窝网络、位置服务 暂不强制，待标准制定 标准出台后

🔑 建议：关键决策点
是否一刀切？
不建议。导线、标准件等低复杂度品类强制 BOM+VSM ROI 低，建议分级管理。
拆解审核由谁负责？
建议由品类采购 + 成本工程师双签，系统记录审核轨迹。
新供应商如何处理？
首次合作可允许"自行拆解"，第 2 次报价起建议升级为 BOM 结构。
供应商不配合怎么办？
建议与供应商评分挂钩，连续 2 次不拆解可考虑降低供应商等级。
