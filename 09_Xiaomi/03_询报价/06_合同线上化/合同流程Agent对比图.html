合同执行流程 · Agent 自动化前后对比

合同签署分 4 步：创建合同 → 填写合同信息 → 发起审批 → 完成签署　|　借助 Agent 精简流程、自动生成、进度可视

🎯 项目目标

构建合同全生命周期在线管理能力，建立统一模板库与签署入口，覆盖起草→审批→签署→归档全链路。

解决合同分散、查找困难、多系统重复维护等痛点。

💡 业务价值

降本提效 模板标准化复用，在线起草审批替代线下流转，缩短签署周期

统一管控 收拢分散签署通道，统一入口

数据贯通 打通集团合同系统与 EP，统一视图、按权限集中查阅

资产沉淀 历史 + 新增结构化入库，多维检索，支撑风险管控

Before 现状 · 人工流程为主
1创建合同
首页待办创建业务完成后生成签署任务，进入申请单
手动填写主体主体 / 金额 / 有效期人工录入
多步跳转创建流程需多页跳转
→
2填写合同信息
下载模板同步定点信息后下载软版
线下修改合同Word 手工调整价格 / 零件
逐份核对人工比对定点价格
→
3发起合同审批
扫描转 PDF 上传Word→PDF 手动转换上传
审批流转点击发布，状态待确认
线下盖章签署大额线下流转盖章
→
4追踪 / 归档
人工查询进度逐条催办、跨系统跳转
逾期靠人工发现无超时预警
分散归档多系统重复维护、检索难
After Agent 协同 · 自动化 + 在线化
1创建合同

→
2填写合同信息
合同草稿自动生成Agent+开发模板智能填充生成草稿，协同开发
→
3发起合同审批
统一签署入口开发收拢分散签署通道
支持电子签 + 线下签开发线上电子签与线下签双模式

→
