五层 AI 能力体系

L1
采购数字员工
整体编排层，作为人机协同核心枢纽——接收采购工程师指令，理解业务目标，协调各层 Agent 协同工作。在关键决策节点（定点会 / 审批 / 定价确认）暂停等待人工确认，输出决策建议供人判断，确保人始终掌握最终决策权。
编排层 · 人机协同枢纽
↓
L2
智能决策层
承载「辅助决策」能力——价格偏离归因、供应商风险评估、成本趋势预测、品类替代推荐。该层依赖底层数据质量（成本因子库、绩效数据、报价历史），随数据底座逐步完善分批上线。
辅助决策 · 随数据底座分批上线
↓
L3
流程执行层
承载「流程提效」能力，将高频重复任务自动化——样件下单、合同生成、问卷解析、定价比对、任务路由。该层对底层数据依赖较低，可基于现有系统能力快速落地，同时在流程运行中自然沉淀数据，为 L2 决策层积累燃料。
⚡ 当前优先 · Q3 重点发力
↓
L4
技能层（Skill）
将确定性业务规则封装为可复用的技能单元——"查询供应商资质"、"比对历史报价"、"生成合同草稿"、"检查定价偏离"——供上层 Agent 调用即执行，无需重复开发。
可复用原子技能
↓
L5
系统能力层（CLI）
原子级接口对接后端业务系统——当前已完成 EP 系统 19 个查询 CLI 的开发，后续按 WAVE 节奏扩展覆盖订单、合同、变更等写入能力，使 Agent 具备完整的系统操作闭环。
19 个 CLI 已完成 · WAVE 持续扩展
🔄
协同模式：五层联动，人机协同决策闭环
L1 数字员工 → L2 智能决策 → L3 流程执行 → L4 技能调用 → L5 CLI/API   |  定点会 / 审批 / 定价确认等人工决策节点由 L1 暂停等待人员确认
当前阶段优先聚焦：流程提效（L3）
1
L2 智能决策层依赖底层数据质量（成本因子库、供应商绩效数据、报价历史基准），当前 数据底座尚未建成，强行上线决策层将导致模型准确率不足、人工修正成本过高
2
L3 流程执行层对底层数据依赖较低，可基于现有系统能力（CLI / API）快速落地。Q3 优先在样件下单、合同生成、任务路由等高频重复场景发力，快速交付可感知的效率提升
3
L3 流程执行过程中自然沉淀业务数据——报价记录、供应商交互日志、订单履约数据——这些沉淀数据将形成高质量训练素材，为后续 L2 决策层分层上线积累燃料
4
采取「以用促建」策略：L3 先跑通流程自动化闭环 → 验证 CLI 覆盖度与稳定性 → 沉淀数据 → L2 按品类 / 场景分批上线决策能力，避免大爆炸式交付风险
