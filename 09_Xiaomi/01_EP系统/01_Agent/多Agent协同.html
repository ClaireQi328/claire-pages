"最优"的四维定义

最优匹配

不是选"最好的供应商"，而是选"最适合这个项目的供应商"——技术能力、质量体系、产能布局、协同意愿与项目需求的契合度最高。

Fit = Σ(Wi × Scorei)，W由品类特性决定

最优成本

不是采购单价最低，而是TCO最低：采购价 + 物流费 + 质量损失 + 模具摊销 + 切换成本 + 未来年降空间。

TCO = Price + Logistics + CoQ + Tooling + Switch + ΔAnnualDown

最优效率

消除等待浪费（进度卡点无人知）、消除重复劳动（手动查/填）、消除信息不对称（各角色各说各话）。

Cycle = ValueAddTime + 0×WaitTime + 0×ReworkTime

最优风险

风险在定点前识别，而非量产后暴露：交付不断供、质量不批退、价格不跳涨、合规不暴雷。

Risk = P(failure) × Impact → 定点前评估使P↓

⬡ AI Native 六大原则 × 采购寻源

原则一
环境持续感知

Agent 7×24实时感知EP/QMS/WMS系统状态，状态变更秒级触发后续动作。

传统：人每天巡检30min → AI Native：实时事件驱动
原则二
动态任务编排

Coordinator根据当前进度动态决定下一步：该催就催，该生成就生成，不走固定脚本。

传统：固定workflow → AI Native：基于状态动态决策
原则三
多维信息聚合

Agent聚合5角色×N系统数据生成统一画像，消除信息孤岛。人不再需要"对齐信息"。

传统：各看各的系统 → AI Native：统一画像一屏可见
原则四
可解释推荐

供应商推荐排序附带每条理由和风险提示。人可以challenge，Agent不替人决策。

传统：经验+直觉 → AI Native：数据驱动+可解释
原则五
长周期学习飞轮

每轮定点的量产表现反哺推荐模型。组织采购智慧不随人员流动丢失，沉淀在模型中持续进化。

传统：经验属人，人走即失 → AI Native：知识属组织
原则六
HITL安全边界

Agent可推荐、可催办、可生成草稿，但不可做定点决策、不可锁价、不可签合同。

传统：全人工 → AI Native：Agent辅助+人决策

! 业务场景 & 痛点

核心痛点

采购寻源定点时需要在EP系统不同功能页面分别查询：SOR任务进度、预选供应商进度、询报价进度、物流确认单进度。所有进度完成后再手动创建定点申请单触发审批。需频繁跨系统/页面查询进度 + 大量线下沟通催办 + 进度完成后手动创建申请单。

涉及流程节点：采购管理流程 5.2.1「全新量产零件定点」中 SOR1发布 → 预选供应商 → RFQ询报价 → 供应商评审 → 物流确认 → 定点推荐与决策

收尾 · 1min — 核心信息

预判 Q&A
Q：Agent查的数据实时性怎么保证？
两种模式并行——每日9点定时巡检保证日级覆盖，EP状态变更事件实时监听保证分钟级响应。Validator校验数据时效性。
Q：如果Agent催错了人怎么办？
Validator校验催办对象匹配性。催办只是飞书消息提醒，不触发系统操作，最坏情况=多收一条消息。
Q：跟EP系统现有的待办提醒有什么区别？
EP待办是单点提醒。我们做的是跨环节端到端进度汇聚——整体完成度、缺什么、对定点时间线的影响。
