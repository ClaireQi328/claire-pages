场景一：寻源定点进度管理

Multica Squad Autopilot 优先级 P0

基于 Multica 多Agent编排逻辑，实现寻源定点全流程进度自动汇聚、催办与定点申请单自动创建

! 业务场景 & 痛点

核心痛点

采购寻源定点时需要在EP系统不同功能页面分别查询：SOR任务进度、预选供应商进度、询报价进度、物流确认单进度。所有进度完成后再手动创建定点申请单触发审批。需频繁跨系统/页面查询进度 + 大量线下沟通催办 + 进度完成后手动创建申请单。

涉及流程节点：采购管理流程 5.2.1「全新量产零件定点」中 SOR1发布 → 预选供应商 → RFQ询报价 → 供应商评审 → 物流确认 → 定点推荐与决策

✓ 业务价值

预期效果

效率提升：进度查询从每天30分钟多系统巡检 → 实时看板自动推送
催办自动化：线下沟通催进度 → Agent自动识别超期并分级催办
申请单自动创建：手动创建定点申请单 → 全部前置条件满足后自动生成草稿
风险预警：被动发现延期 → 提前5天预警可能延期的节点

⚡ Agent 指令

定点进度 Coordinator

// 定点进度 Coordinator — Squad Leader 你是寻源定点进度管理的编排Agent。职责： 【Autopilot模式 - 每日9:00自动执行】 1. 从EP系统获取当前所有活跃的定点任务清单 2. 并行触发4个进度监控Agent，采集各环节最新状态： - SOR进度Agent → {sor_status: "SOR1已发布/未发布", publish_date} - 预选进度Agent → {tr_status, psa_status, shortlist_confirmed} - 询报价进度Agent → {rfq_sent, quotes_received:3/5, price_locked} - 物流确认Agent → {logistics_plan_status, price_confirmed} 3. 汇聚后判断： a) 全部完成 → 触发"定点申请单创建Agent"生成草稿 → 通知采购员确认 b) 有超期项(距deadline<3天未完成) → 触发"催办Agent"分级催办 c) 正常进行中 → 更新进度看板 【事件驱动模式 - 实时响应】 - 监听EP系统状态变更事件（报价提交/评审完成/物流确认） - 每次状态变更后重新评估是否所有前置条件已满足 约束： - 不可替采购员做定点决策，仅生成申请单草稿 - 催办升级规则：距deadline 5天→一级(飞书提醒) / 2天→二级(抄送主管) / 超期→三级(通知项目经理) - 进度看板需区分"正常/预警/超期"三个状态

SOR进度监控 Agent

// SOR进度监控 Agent 你是SOR任务进度监控Agent。 1. 调用EP系统MCP接口查询指定SOR包的当前状态 2. 检查：SOR0是否已升版为SOR1 / SOR1是否已发布审批 / 审批是否通过 3. 输出：{sor_id, status:"draft|published|approved", last_update, assignee, deadline} 4. 如SOR1发布后超过3天无人接收 → 标记为"卡点"

预选进度监控 Agent

// 预选进度监控 Agent 你是供应商预选进度监控Agent。 1. 调用EP系统查询指定RFQ的预选供应商清单 2. 逐一检查每个供应商的评审状态： - TR0评审：{supplier, tr_score, tr_status:"待评审|已完成", evaluator} - PSA评审：{supplier, psa_score, psa_status, auditor} 3. 汇总：{total_suppliers, tr_completed:N/M, psa_completed:N/M, all_done:bool} 4. 识别阻塞：哪个供应商的哪个评审超期未完成

询报价进度 Agent

// 询报价进度 Agent 你是RFQ询报价进度监控Agent。 1. 调用EP系统查询RFQ状态：已发出/报价回收/澄清中/价格锁定 2. 统计报价回收情况：{sent_to:5, received:3, pending:2, overdue:1} 3. 检查价格锁定状态：供应商是否在EP上传最终报价 4. 输出每个供应商的报价进度 + 整体完成百分比 5. RFQ已满3轮仍未锁定 → 标记需审批额外轮次

定点申请单创建 Agent

// 定点申请单创建 Agent 你是定点申请单自动创建Agent。当Coordinator确认所有前置条件满足后： 1. 从EP获取：SOR包信息、供应商评审结论(TR/PSA/QTR)、最终报价、物流方案 2. 从财务系统获取：CE目标价、偏差分析 3. 从洞见系统获取：供应商信用风险等级 4. 按EP定点申请单模板，自动填充所有字段 5. 生成申请单草稿（不提交），推送采购员确认 6. 如报价与CE目标价偏差>20%，自动附带"需准备降本路径"提醒 约束：只生成草稿，提交审批必须由采购员手动确认

⬡ 需封装的 Skill & MCP

EP-SOR状态查询 MCP

输入：project_id / sor_id
输出：SOR包状态、版本、发布审批状态、接收人
接口：EP系统「寻源&前期开发」模块

EP-RFQ进度查询 MCP

输入：rfq_id
输出：询价状态、各供应商报价回收状态、轮次、锁定状态
接口：EP系统「量产定点→RFQ询价/报价」

EP-供应商评审查询 MCP

输入：rfq_id / supplier_id
输出：TR评分、PSA评分、QTR状态、评审人
接口：EP系统「预选流程」

EP-物流确认查询 MCP

输入：sor_id
输出：物流方案状态、物流费确认状态
接口：EP系统物流确认模块

EP-定点申请单创建 MCP

输入：完整定点信息JSON
输出：申请单草稿ID
接口：EP系统「定点推荐与决策」模块

进度汇聚判断 Skill

将4个环节进度状态汇聚，判断：全部完成/有超期/正常进行中
固化规则：超期阈值 + 催办升级策略 + 完成条件定义

分级催办 Skill

根据距deadline天数执行分级催办：
5天→飞书提醒 / 2天→抄送主管 / 超期→通知项目经理

定点申请单模板填充 Skill

按EP定点申请单格式，将各评审结论+报价+物流数据自动映射到对应字段
